# 分类规范化设计

**目标**：让分类体系自动收口，避免「脑科学 / 神经科学 / 脑科学/神经科学」这类近义标签继续并行增长。

**结论**：采用严格规则模式。模型仍可自由给出分类建议，但真正写入数据库前，所有分类都要先经过一层规范化映射，再生成统一的 `categories` 与 `categoryKeys`。

## 设计

系统增加一份可维护的分类规范表，定义每个主分类的标准写法，以及需要吸收的别名、变体和路径写法。第一版先覆盖当前最明显的冲突，尤其是脑科学相关标签。

规范化发生在三处：

1. 新资料导入时，LLM 返回的分类先被规范化，再写入 Dexie 和 SQLite。
2. 手动「自动归类」时，API 返回的分类先被规范化，再落库。
3. 数据库升级时，批量扫描已有概念，把旧的分裂标签回写成规范后的结果。

## 规则

第一版采用严格模式：

- 主类只保留一个入口，例如「脑科学」
- 「神经科学」「脑科学/神经科学」这类写法全部并入主类
- 二级分类只保留真正能区分内容方向的词，不再保留对主类的重复表述

这样做的目的不是消灭层级，而是避免同义词伪装成层级。

## 实现边界

这不是一次性写死所有世界上的标签。系统允许新增真正新的主领域，但对于已知高频近义词，会自动收口到主名称。后续只要继续往规则表补映射，就能扩展自动化覆盖面。

# 设置页重构：分 Tab + 字号调整

**日期**：2026-04-27
**状态**：设计完成，待实施

---

## 1. 背景

设置页（`SettingsDrawer`）当前以一个长滚动列表呈现全部设置项（LLM 配置、访问保护、Wiki 维护、数据管理），内容越来越多，可读性下降。同时用户需要调整 Wiki 概念详情页和资料详情页的正文字号大小。

本次改动包含两个目标：

1. **设置页分 Tab 重构** — 将设置项按功能分组，提供清晰的导航
2. **字号调整功能** — 新增阅读字号设置，统一影响详情页正文

## 2. 设计决策

| 决策点        | 选择                                       | 理由                                      |
| ------------- | ------------------------------------------ | ----------------------------------------- |
| Tab 数量      | 3 个：通用 / 模型 / 数据                   | 每个 Tab 内容适中，不多不少               |
| 桌面端导航    | 左右分栏（左侧导航列表 + 右侧内容）        | 信息密度高，符合 Notion/Obsidian 惯例     |
| 移动端导航    | 分段控制 (Segmented Control)               | 紧凑，和现有颜色模式/首页样式控件风格一致 |
| 字号控件      | 预设档位按钮组（微信读书风格）             | 所见即所得，操作直观                      |
| 字号作用范围  | 统一调整，同时影响 Wiki 详情页和资料详情页 | 简洁，减少认知负担                        |
| 字号档位      | 5 档：小/较小/中/较大/大                   | 覆盖绝大多数需求                          |
| 访问保护归属  | 放在「模型」Tab                            | API Key 和 Admin Token 都是密钥类配置     |
| Lint 体检归属 | 放在「数据」Tab                            | 和数据管理同属维护类操作                  |

## 3. Tab 内容分配

### Tab 1：通用

外观偏好与阅读体验设置。

| 设置项   | 控件类型                             | 说明                  |
| -------- | ------------------------------------ | --------------------- |
| 颜色模式 | 分段控制（浅色/深色/系统）           | 保持现有实现          |
| 正文字号 | 预设档位按钮组（小/较小/中/较大/大） | **新增**，详见第 4 节 |
| 首页样式 | 分段控制（动态流/知识库）            | 保持现有实现          |

### Tab 2：模型

AI 模型配置与访问安全。

| 设置项   | 控件类型                           | 说明                      |
| -------- | ---------------------------------- | ------------------------- |
| LLM 配置 | 文本输入 + 预设快捷按钮 + 高级展开 | 保持现有实现              |
| 访问保护 | 密码输入 + 保存/清除按钮           | 从独立 section 移入此 Tab |

### Tab 3：数据

Wiki 维护工具与数据管理。

| 设置项           | 控件类型            | 说明         |
| ---------------- | ------------------- | ------------ |
| Lint · Wiki 体检 | 按钮触发 + 结果列表 | 保持现有实现 |
| 载入示例 Wiki    | 按钮 + 二步确认     | 保持现有实现 |
| 清空所有数据     | 危险按钮 + 二步确认 | 保持现有实现 |

## 4. 字号调整功能

### 4.1 档位定义

| 档位 | 标签       | 正文字号 | 行高 |
| ---- | ---------- | -------- | ---- |
| `xs` | 小         | 14px     | 1.6  |
| `sm` | 较小       | 15px     | 1.6  |
| `md` | 中（默认） | 16px     | 1.6  |
| `lg` | 较大       | 18px     | 1.6  |
| `xl` | 大         | 20px     | 1.6  |

### 4.2 作用范围

字号仅影响以下两个详情页的 **正文内容区域**（`.prose` / `.concept-body-prose` 以及 SourceDetail 的编辑器区域）：

- `ConceptDetail` — Wiki 概念详情页的 `<Prose>` 正文
- `SourceDetail` — 资料详情页的 contentEditable 编辑区域

**不受影响的部分**：

- 导航栏、Tab 栏、Header 区域
- 列表页（WikiView、SourcesView）
- 设置页本身
- Meta 信息行、标签芯片
- 详情页的页面级标题（concept.title / source.title 渲染的 `<h1>`）、kicker、intro 段落

> 注意区分：页面级标题不受影响，但 Markdown 正文内渲染出的 h1/h2/h3 会跟随字号缩放（见 4.3 节）。

### 4.3 标题缩放

正文字号变化时，Markdown 渲染后的标题通过 CSS 比例系数自动缩放：

| 元素                | 缩放系数         |
| ------------------- | ---------------- |
| h1                  | 基础字号 × 1.75  |
| h2                  | 基础字号 × 1.5   |
| h3                  | 基础字号 × 1.25  |
| p / li / blockquote | 基础字号 × 1.0   |
| code                | 基础字号 × 0.875 |

### 4.4 技术实现

**状态管理**：在现有 Zustand store（`lib/store.ts`）中新增 `fontSize` 状态和 `setFontSize` / `hydrateFontSize` 方法，与 `colorMode` 的实现模式保持一致。

```typescript
export type FontSize = 'xs' | 'sm' | 'md' | 'lg' | 'xl';

// 在 AppState 中新增
fontSize: FontSize;
setFontSize: (size: FontSize) => void;
hydrateFontSize: () => void;
```

**存储**：`localStorage` key 为 `compound-font-size`，与现有 `compound-color-mode`、`compound-home-style` 命名模式一致。

**CSS 变量**：在 `globals.css` 中定义 `--prose-font-size` 变量，默认值 `16px`。`setFontSize` 方法在设置 store 和 localStorage 的同时，调用 `document.documentElement.style.setProperty('--prose-font-size', value)` 更新 CSS 变量。

**Prose 样式**：修改 `.prose` 相关 CSS，让正文字号引用 `var(--prose-font-size)` 而非硬编码值，标题通过 `calc()` 基于该变量缩放。

**防闪烁（FOUC）**：在 `layout.tsx` 的 `<head>` 中添加同步脚本，启动时从 `localStorage` 读取字号偏好并立即设置 CSS 变量，和现有的颜色模式防闪烁脚本放在一起。

**初始化**：在 `app/page.tsx` 的顶层 effect 中调用 `hydrateFontSize()`，和 `hydrateColorMode()` 并列。

## 5. 设置页响应式布局

### 5.1 桌面端（≥ 768px）

```
┌─────────────────────────────────────────────┐
│  Compound · 设置                    [关闭]  │
├──────────────┬──────────────────────────────┤
│              │                              │
│  🎨 通用     │   通用设置                    │
│  ✨ 模型     │   外观、阅读偏好和首页风格      │
│  🗄️ 数据    │                              │
│              │   ┌ 颜色模式 ─────────────┐  │
│              │   │ [浅色] [深色] [系统]   │  │
│              │   └──────────────────────┘  │
│              │                              │
│              │   ┌ 正文字号 ─────────────┐  │
│              │   │ [小][较小][中][较大][大]│  │
│              │   └──────────────────────┘  │
│              │                              │
│              │   ┌ 首页样式 ─────────────┐  │
│              │   │ [动态流] [知识库]      │  │
│              │   └──────────────────────┘  │
│              │                              │
├──────────────┴──────────────────────────────┤
```

左侧导航宽度约 180px，使用图标+文字标签，当前选中项高亮背景。右侧内容区显示对应 Tab 的设置项。

### 5.2 移动端（< 768px）

```
┌─────────────────────────────────┐
│  ═══ (handle)                   │
│  Compound · 设置        [关闭]  │
│                                 │
│  ┌──────────────────────────┐   │

│  └──────────────────────────┘   │
│                                 │

│  [☀️ 浅色] [🌙 深色] [⚙️ 系统]  │

│  [小] [较小] [中] [较大] [大]    │
```

使用 CSS media query `@media (min-width: 768px)` 切换布局，与现有应用的桌面/移动双布局断点一致。

### 6.1 文件变更清单

| 文件                                 | 变更类型 | 说明                                                                                               |

### 6.2 SettingsDrawer 内部组件拆分

当前 `SettingsDrawer.tsx` 是一个约 570 行的单文件组件。重构后按 Tab 拆分子组件：

```
components/
  SettingsDrawer.tsx          — 壳子：modal overlay + header + Tab 导航 + 路由
  settings/
    GeneralTab.tsx            — 颜色模式 + 字号 + 首页样式
    ModelTab.tsx              — LLM 配置 + 访问保护
    DataTab.tsx               — Wiki 维护 + 数据管理
    FontSizeSelector.tsx      — 字号预设档位选择器（可复用）
```

## 7. 交互细节

- **Tab 切换**：无动画，即时切换内容。默认打开「通用」Tab。
- **字号切换**：实时生效，无需保存按钮。切换后详情页正文字号立即更新。
- **记忆状态**：设置页关闭后再打开，记住上次选中的 Tab（session 级别，存在组件 state 中，不持久化）。
- **桌面端左侧导航**：hover 时显示浅色高亮背景，当前选中项用更深的高亮背景。
- **移动端分段控制**：和现有的颜色模式分段控制视觉风格完全一致（背景色、圆角、阴影）。

## 8. 不做的事情

- **不做**字号实时预览弹窗（在设置页内选择即可，打开详情页自然看到效果）
- **不做**标题和正文分开调整字号（一个控件统一调整，标题通过比例系数跟随）
- **不做**自定义字号数值输入（5 档预设足够）
- **不做**设置页搜索功能（3 个 Tab 共约 7 个设置项，无需搜索）
- **不做**设置页过渡动画（简单切换即可）

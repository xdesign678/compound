# 仓库协作约定

- **以后所有修改一律直接在 `main` 分支上进行，不要新建分支，也不要走 PR 流程。**
- 开始工作前先确认当前分支是 `main`（`git rev-parse --abbrev-ref HEAD` 应返回 `main`）；若不在 `main`，先 `git checkout main` 再修改。
- 完成改动后直接 `git commit` 并 `git push origin main`，不要创建功能分支或 PR。
- 仅在用户明确要求时，才可以新建分支或开 PR。

## Naming conventions

Naming rules are enforced via `@typescript-eslint/naming-convention` in `.eslintrc.json`.

| Category                    | Convention                     | Examples                            |
| --------------------------- | ------------------------------ | ----------------------------------- |
| Functions & methods         | camelCase                      | `extractSearchTerms`, `postJSON`    |
| React components            | PascalCase                     | `NoteEditor`, `SettingsDrawer`      |
| Next.js route handlers      | UPPER_CASE (HTTP method names) | `GET`, `POST`                       |
| Variables                   | camelCase                      | `modalOpen`, `startRef`             |
| Constants (module-level)    | UPPER_CASE                     | `MAX_ENTRIES`, `GC_MIN_INTERVAL_MS` |
| Component variables (const) | PascalCase                     | `const IngestModal = lazy(…)`       |
| Type aliases & interfaces   | PascalCase                     | `SourceType`, `LlmConfig`           |
| Enum members                | PascalCase or UPPER_CASE       | `Active`, `MAX_RETRIES`             |
| Parameters                  | camelCase, leading `_` allowed | `req`, `_ctx`                       |
| Private class members       | camelCase, leading `_` allowed | `_store`, `_timer`                  |
| Files (utilities)           | kebab-case                     | `api-client.ts`, `server-logger.ts` |
| Files (components)          | PascalCase                     | `SwipeBack.tsx`, `NoteEditor.tsx`   |

## Automated API documentation

- HTTP API reference lives at `docs/api-reference.md` and is generated by
  `scripts/generate-api-docs.mjs` from `app/api/**/route.ts`.
- Run `npm run docs:api` to regenerate after adding/removing/renaming
  routes or editing JSDoc on a handler. Run `npm run docs:api:check` to
  verify the file is up to date (CI runs this automatically).
- Documenting an endpoint means adding a `/** ... */` JSDoc block directly
  above the exported `GET` / `POST` / etc. handler in its `route.ts`.
- A push to `main` that touches `app/api/**` or the generator triggers
  `.github/workflows/docs-autogen.yml`, which regenerates and commits
  `docs/api-reference.md` on the agent's behalf.

## Build performance tracking

- 运行 `npm run build:measure`（即 `node scripts/measure-build.mjs`）来执行 `next build` 并记录耗时、`.next/cache` 命中情况以及 `.next/static` 产物大小。
- 指标会写入 `tmp/build-metrics.json`（已在 `.gitignore` 中忽略），脚本的退出码与 `next build` 保持一致。
- 在 GitHub Actions 中，`.next/cache` 通过 `actions/cache@v4` 按 `package-lock.json` 与源码哈希缓存，命中即可加速重复构建；指标文件会作为 `build-metrics` 产物上传，并写入当次 job 的 Step Summary。

## Module boundary enforcement

Architectural boundaries between server-side and client-side code are enforced via the **import/no-restricted-paths** ESLint rule configured in `.eslintrc.json`.

**Server-only modules** (use Node.js APIs such as better-sqlite3, node:fs, node:crypto, node:net, node:dns, node:async_hooks, process.env) must NOT be imported from client-side code (components, pages). These include files matching `lib/server-db.ts`, `lib/server-auth.ts`, `lib/server-ingest.ts`, `lib/server-logger.ts`, `lib/gateway.ts`, `lib/embedding.ts`, `lib/repair-worker.ts`, `lib/rate-limit.ts`, `lib/request-guards.ts`, `lib/request-context.ts`, `lib/review-queue.ts`, `lib/analysis-worker.ts`, `lib/model-runs.ts`, `lib/wiki-db.ts`, `lib/wiki-chunk.ts`, `lib/wiki-compiler.ts`, `lib/sync-observability.ts`, `lib/github-sync-runner.ts`, `lib/github-sync.ts`, `lib/logging.ts`, `lib/circuit-breaker.ts`, and `lib/observability/*.ts`.

- Client-side code should call `app/api/**` routes instead of importing server modules directly.
- API routes must NOT import `lib/db.ts` (Dexie/IndexedDB) or `lib/store.ts` (Zustand) — use `lib/server-db.ts` (better-sqlite3) for server-side data access.

**Client-safe modules** (browser-compatible, usable from both client and server): `lib/db.ts`, `lib/store.ts`, `lib/api-client.ts`, `lib/cloud-sync.ts`, `lib/format.ts`, `lib/types.ts`, `lib/haptic.ts`, `lib/responsive.ts`, `lib/utils.ts`, `lib/llm-config.ts`, `lib/seed.ts`, `lib/admin-auth-client.ts`, `lib/trace-client.ts`, `lib/github-sync-ui.ts`, `lib/github-sync-poll.ts`, `lib/ask-suggestions.ts`, `lib/categorize-status.ts`, `lib/concept-body-format.ts`, `lib/content-status.ts`, `lib/category-normalization.ts`, `lib/model-history.ts`, `lib/snapshot-merge.ts`, `lib/ingest-core.ts`, `lib/obsidian-import.ts`.

## Incident response runbooks

- 事故处理入口在 `runbooks/README.md`。生产 503、访问保护、GitHub 同步卡住、LLM 网关失败、SQLite 持久化异常，先按对应 playbook 排查。
- 处理事故时先记录用户影响、第一条真实错误、最近配置/部署变化和已执行动作；不要为了恢复速度跳过访问保护或直接改生产 SQLite。

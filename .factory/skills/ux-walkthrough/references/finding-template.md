# 发现卡片模板

走查 Agent 输出发现时使用此格式。

---

## 标准格式

```markdown
### [严重程度] 发现标题（一句话描述问题）

| 字段 | 值 |
|------|-----|
| **编号** | `[维度]-[序号]` (如 A-01, B-03, E-07) |
| **严重程度** | Critical / Major / Minor / Enhancement |
| **Nielsen 原则** | H1-H10 中的具体条目 |
| **页面** | 具体页面/Tab + 视口尺寸 |
| **状态** | 触发此问题的前置条件（正常/空/离线/错误/首次使用） |

**场景还原：** 用户在做什么操作时会遇到这个问题。一句话。

**现状：** 当前的实际行为是什么。

**期望：** 理想情况应该是什么样。

**用户影响：** 这对用户意味着什么（用非技术语言）。

**修复建议：** 具体怎么改。如果是代码修改，给出文件路径和大致思路。

**截图/快照：** （如有）Playwright 截图文件名。
```

---

## 严重程度判定标准

| 级别 | 判定条件 | 典型例子 |
|------|---------|---------|
| **Critical** | 阻断核心任务完成 OR 导致数据丢失 OR 页面崩溃 | 按钮功能指向错误页面；保存失败但没提示；JS 错误导致白屏 |
| **Major** | 造成显著困惑 OR 工作流中断 OR 需额外步骤变通 | Tab 名和内容不匹配；3s+ 无加载提示；错误信息看不懂 |
| **Minor** | 可感知的摩擦但不阻碍完成任务 | 间距不统一；缺 tooltip；暗色模式小瑕疵 |
| **Enhancement** | 当前能用，做了会更好 | 添加快捷键；阅读进度条；触觉反馈 |

## 大白话版本指南

生成面向非技术读者的可视化报告时，每个发现需要：

1. **标题用动词开头** — "打开首页后页面自己跳走了"而非"首页自动重定向"
2. **场景还原用"你会遇到"** — "你打开 App，正准备看知识库，结果 2 秒后页面自己跳到了设置"
3. **类比生活场景** — "就像你打开微信想看聊天，结果 App 自己跳到了设置页面"
4. **修复建议说人话** — "让搜索按钮打开搜索框"而非"修复 onClick 事件绑定"
5. **不出现术语** — 不用 H1/H2、Nielsen、token、CSS variable 等词

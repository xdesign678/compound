# Nielsen 10 大可用性原则 — 走查清单

每个走查 Agent 按分配的原则检查以下项目。

---

## H1 系统状态可见性

### 加载状态
- [ ] 每个数据获取 100ms 内显示加载指示（spinner/skeleton）
- [ ] Skeleton 形状匹配实际内容（不只是通用矩形块）
- [ ] 内容替换 skeleton 时无布局偏移
- [ ] 超过 2s 的操作显示进度而非只有 spinner
- [ ] 过期数据与新鲜数据有视觉区分

### 同步/保存状态
- [ ] 当前同步状态始终可见（已同步/同步中/离线/错误）
- [ ] 显示"上次同步于 X 分钟前"
- [ ] 编辑有可见的保存确认（"保存中..." vs "已保存 ✓"）
- [ ] 保存失败立即标记（不能静默失败）

### 离线指示
- [ ] 断网后 1 秒内显示离线状态
- [ ] 离线指示器持久显示（不是一闪而过的 toast）
- [ ] 用户知道哪些功能离线可用、哪些不可用
- [ ] 恢复网络后有视觉确认

---

## H2 系统与现实匹配

### 语言与术语
- [ ] 核心概念在全 App 中叫法统一（不能一处叫"笔记"一处叫"概念"）
- [ ] 不向用户暴露技术术语（IndexedDB、sync conflict、HTTP 401）
- [ ] 日期用本地化格式（"3 分钟前"而非 ISO timestamp）
- [ ] Tab/按钮名称与实际内容匹配

### 图标与隐喻
- [ ] 图标含义直观可识别
- [ ] 不同类型的内容有视觉区分（PDF vs URL vs 笔记）

---

## H3 用户控制与自由

### 撤销/重做
- [ ] 删除操作可在宽限期内撤销
- [ ] 编辑内容可回退到之前的版本
- [ ] 撤销入口可发现（toast with action button）

### 导航自由
- [ ] 返回按钮始终可用（无死路）
- [ ] 弹窗/抽屉可通过 ×、背景点击、Escape 三种方式关闭
- [ ] 强制流程（引导、设置向导）可以退出
- [ ] 深度链接可用（URL 可分享到具体内容）

---

## H4 一致性与标准

### 设计 Token 遵从度
- [ ] 颜色全部引用 CSS 变量（无硬编码 hex/rgba）
- [ ] 间距使用统一 scale（4/8/12/16/24/32px）
- [ ] 圆角在同类组件间一致
- [ ] 阴影形成连贯的层级系统
- [ ] 字号遵循 type scale

### 组件一致性
- [ ] 按钮高度/padding 全局统一
- [ ] 卡片 padding/radius/shadow 一致
- [ ] 列表项高度/对齐一致
- [ ] 输入框高度/边框/focus 态一致
- [ ] 弹窗背景/动画/padding 一致

### 平台规范
- [ ] 移动端用底部抽屉（非桌面弹窗）
- [ ] 触摸目标 ≥ 44px
- [ ] hover 态仅在非触屏设备显示

---

## H5 错误预防

### 表单校验
- [ ] 提交前有内联校验
- [ ] 字符限制主动显示（不只在超限时报错）
- [ ] 重复内容/名称被提前捕获
- [ ] 破坏性操作（删除）有确认对话框

### 数据保护
- [ ] 导航时警告未保存更改
- [ ] 浏览器关闭/刷新有 beforeunload 保护
- [ ] 意外手势（滑动返回）不会丢失编辑状态

---

## H6 识别而非回忆

### 位置感知
- [ ] 当前位置始终清晰（活跃 Tab 高亮、面包屑）
- [ ] 最近访问的内容容易找到
- [ ] 相关内容在需要时可见（不用搜索才能发现）
- [ ] 搜索随输入显示建议

---

## H7 灵活与高效

### 快捷键
- [ ] 常用操作有快捷键（搜索、新建、导航）
- [ ] 快捷键可发现（tooltip 或帮助面板）
- [ ] 不与浏览器/OS 快捷键冲突

### 高级功能
- [ ] 命令面板可用且覆盖常用操作
- [ ] 排序/筛选设置跨会话持久
- [ ] 批量操作可用（多选、批量标签）

---

## H8 美学与极简

### 视觉层次
- [ ] 每屏主操作明确突出
- [ ] 信息密度适合对应视口
- [ ] 留白有目的（不是剩余空间）

### 排版
- [ ] 阅读内容用 serif 字体
- [ ] UI 元素用 sans 字体
- [ ] 行高适合阅读（body 1.5-1.7）
- [ ] 移动端字号 ≥ 14px

### 动效
- [ ] 微交互过渡 ≤ 300ms
- [ ] 动画有目的（引导注意力、表示关系）
- [ ] reduced-motion 偏好被尊重

---

## H9 帮助用户识别/诊断/恢复错误

### 错误消息
- [ ] 用户语言（非技术术语/错误码/堆栈）
- [ ] 说明发生了什么 AND 下一步该做什么
- [ ] 区分用户可修复 vs 系统错误
- [ ] 具体（非笼统"出了点问题"）

### 恢复
- [ ] 所有失败操作有重试按钮
- [ ] 网络恢复后自动重试（带指示）
- [ ] 部分失败不阻塞整个 UI

---

## H10 帮助与文档

### 引导
- [ ] 首次用户看到价值主张 + 快速开始
- [ ] 空状态引导第一步操作
- [ ] 渐进式披露（不在第一天淹没用户）

### 上下文帮助
- [ ] 复杂功能有内联说明
- [ ] 非显而易见的图标有 tooltip
- [ ] 有"了解更多"链接

---

## 移动端专项（H3+H4+H7 移动视角）

### 触摸与手势
- [ ] 所有交互元素 ≥ 44×44px 触摸目标
- [ ] 无 hover 依赖的功能
- [ ] 滑动手势有视觉暗示（渐变遮罩/箭头）
- [ ] 下拉刷新在合适的页面可用

### 布局
- [ ] 无水平滚动溢出
- [ ] 输入框聚焦不破坏布局（iOS 键盘推挤）
- [ ] 固定底部导航不遮挡内容
- [ ] 安全区域 inset 被尊重（刘海/Home 指示器）

### PWA
- [ ] 应用壳从缓存即时加载
- [ ] 之前浏览的内容离线可用
- [ ] SW 更新通知非侵入式

# 视觉评测执行 Prompt

将此 prompt 交给有图片识别能力的代理执行（Gemini 3 Pro / Claude）。

---

## 背景

`claude-design-style` 技能已通过 L1 代码分析评测（`run_eval.py`，检查 CSS token 正确性）。
本轮是 **L2 视觉渲染评测** -- 用 browser 截图 + 多模态模型判断页面实际渲染效果是否符合 Anthropic 设计风格。

## 参考标准

Anthropic/Claude 官网的视觉特征（你需要记住这些，截图评分时对照）：

1. **配色**：米白背景 `#faf9f5`（不是纯白），暖灰文字（不是纯黑），无渐变，无鲜艳色块
2. **字体**：正文 serif（Lora/Georgia 类），标题 sans-serif（Geist/Inter 类），代码 monospace
3. **按钮**：深色填充 `#0f0f0e`，微圆角（约 7.5px，不是药丸形），hover 变暗灰
4. **间距**：大量留白，内容区窄（~640px），呼吸感强
5. **卡片**：白色背景，细微边框（几乎看不见），轻阴影
6. **导航栏**：约 68px 高，简洁，Logo 左 + 链接居中/右 + CTA 按钮
7. **整体气质**：文学感、杂志感、高端阅读体验，安静克制，不花哨
8. **暗色模式**：深暖灰背景 `#1a1a18`（不是纯黑），文字 `#ece9e1`，按钮反转为浅色

## Step 2: 逐个截图

对每个测试文件执行以下流程。需要一个本地 HTTP 服务器来正确渲染 HTML。

### 2.1 启动服务器

```bash
cd ~/.claude/skills/claude-design-style/eval/tests
python3 -m http.server 8765 &
```

### 2.2 截图矩阵

对每个文件 T01-T10，截取以下截图：

| 截图 ID             | 视口宽度 | 模式 | 说明                                     |
| ------------------- | -------- | ---- | ---------------------------------------- |
| `{T}-desktop-light` | 1440px   | 亮色 | 桌面端默认视图                           |
| `{T}-mobile-light`  | 375px    | 亮色 | 手机端视图                               |
| `{T}-desktop-dark`  | 1440px   | 暗色 | 桌面端暗色（仅 T01/T03/T04/T06/T07/T10） |
| `{T}-mobile-dark`   | 375px    | 暗色 | 手机端暗色（仅 T01/T03/T04/T06/T07/T10） |

### 2.3 截图流程（每个文件）

```
1. browser navigate http://localhost:8765/{filename}
2. 截图 → screenshots/{T}-desktop-light.png
3. 调整视口到 375px 宽度（如果 browser 支持，或用 DevTools）
4. 截图 → screenshots/{T}-mobile-light.png
5. 如果是暗色模式文件（T01/T03/T04/T06/T07/T10）：
   a. browser act "点击暗色模式切换按钮"（或 browser console exec "document.documentElement.classList.add('dark')"）
   b. 恢复 1440px 视口
   c. 截图 → screenshots/{T}-desktop-dark.png
   d. 调整到 375px
   e. 截图 → screenshots/{T}-mobile-dark.png
   f. browser console exec "document.documentElement.classList.remove('dark')"
6. 继续下一个文件
```

如果 browser 工具不支持调整视口宽度，可以只截桌面端（1440px），跳过手机端截图。

## Step 3: 视觉评分

看每张截图，按以下 8 个维度打分（每项 0-10 分）。

### 评分维度

| #   | 维度           | 说明                                                                     | 满分标准                                 |
| --- | -------------- | ------------------------------------------------------------------------ | ---------------------------------------- |
| V1  | 配色准确度     | 背景是否米白而非纯白？文字是否暖灰而非纯黑？有无渐变/鲜艳色块？          | 背景接近 #faf9f5，无纯黑纯白，无渐变     |
| V2  | 字体系统       | 正文是否 serif？标题是否 sans？代码块是否 mono？视觉上能看出字体层级吗？ | 一眼能看出 serif 正文 + sans 标题的对比  |
| V3  | 按钮与交互元素 | 按钮是否深色填充 + 微圆角？有无药丸形按钮？Clay 色有没有用在按钮上？     | 深色按钮、7-8px 圆角、无药丸、无渐变按钮 |
| V4  | 留白与呼吸感   | 元素之间间距是否充足？内容是否过于拥挤？是否有"杂志感"的留白？           | 大方留白，内容不拥挤，阅读舒适           |
| V5  | 卡片与容器     | 卡片是否白底 + 细边框/轻阴影？有无粗边框或重阴影？圆角是否适度（~8px）？ | 干净卡片，轻阴影，适度圆角               |
| V6  | 整体一致性     | 页面各部分视觉风格是否统一？有无"拼接感"或风格冲突？                     | 从头到脚风格统一，像一个品牌             |
| V7  | 暗色模式质量   | （仅暗色截图）是否深暖灰而非纯黑？按钮是否反转为浅色？对比度是否舒适？   | 暖灰底，文字清晰，按钮反转正确           |
| V8  | 移动端适配     | （仅手机截图）布局是否正确折叠？文字是否可读？按钮是否足够大？有无溢出？ | 单列布局，文字不小于 14px，无横向溢出    |

### 评分规则

- 每张截图只评相关维度（桌面亮色评 V1-V6，暗色加评 V7，手机加评 V8）
- 同一测试的多张截图取各维度最低分（短板原则）
- 最终每个测试得到 V1-V8 各一个分数（0-10）

### 评分时的内心对话模板

看到每张截图时，按这个顺序自问：

1. "第一印象像 Anthropic 官网吗？" -- 如果像给 V6 高分
2. "背景色对吗？" -- 应该是微微发黄的白，不是纯白
3. "能看出 serif 正文吗？" -- 正文应该是衬线体
4. "按钮什么形状？" -- 应该是矩形微圆角，不是药丸
5. "留白够吗？" -- Anthropic 风格留白很多
6. "有没有刺眼的颜色？" -- 不应该有渐变、鲜艳色块、纯黑大面积
7. "暗色模式是暖色调吗？" -- 不是冷灰/纯黑
8. "手机端布局正常吗？" -- 单列、不溢出、按钮够大

## 重要约束

1. **每张截图都必须实际看过再打分** -- 不能根据文件名或代码猜测
2. **评分必须基于视觉印象** -- 不是看代码，是看渲染后的页面
3. **对照标准是 Anthropic 官网** -- 如果你对 Anthropic 官网不熟悉，先 `browser navigate https://www.anthropic.com` 看一眼，建立视觉基准
4. **不修改任何文件** -- 这是纯评测，只读取 + 截图 + 写报告
5. **截图失败时记录原因** -- 如果某个页面打不开或渲染异常，在报告中记录，评分标为 N/A

为什么重要

从审计到修复，闭环交付

此前的审计报告发现了 54 条 UX 问题，涵盖 7 个 Critical、19 个 Major、18 个 Minor 和 10 个 Enhancement。这些问题从键盘快捷键崩溃到 CSS 中 111 个 !important 的技术债，严重影响了产品的可用性和可维护性。

做了什么

核心修复详情

键盘交互修复

Escape 键分层关闭 — 实现优先级链，没有任何东西打开时 Escape 不做任何事，彻底解决意外跳转问题
Cmd+K 崩溃修复 — 添加 clampedIndex 计算和全链路 bounds check，防止 useDeferredValue 延迟导致空数组访问崩溃

错误处理升级

错误卡片化 — /sync 和 /review 页面根据状态码（401/403/500）显示不同图标和中文消息，附带重试和返回按钮
问答错误友好化 — 原始 JSON 错误信息折叠到"查看详情"，添加"重新提问"一键重试
中文 404 页面 — 品牌风格的 not-found 页面，包含返回首页和搜索知识库两个入口

组件功能补全

概念详情操作按钮 — "标记有误"写入审核日志，"删除"带确认对话框防止误操作
Toast 全局化 — 从 page.tsx 移到 layout.tsx，所有路由页面（/sync、/review、/ask）均可显示通知

CSS 设计系统重构

arXiv 2606.28270Agent 安全

Agent-Native Immune System

研究问题
带记忆、工具和多 agent 协作的系统，传统聊天安全边界不够用了。
核心方法
提出六层 Immune Tower，把防御机制嵌进 agent 的认知和执行循环。
为什么重要
模型价值观正确，不等于工具调用不会被劫持。
局限
更像架构框架，产品效果仍需要标准协议和大规模评测。
arXiv 2606.28153越狱机制

Robust Harmful Features

研究问题
越狱提示到底改动了模型内部哪些安全相关部件。
核心方法
分析注意力头，区分被压制的头和仍保持安全激活的头。
为什么重要
如果危险特征仍存在，产品可尝试读取内部激活做实时检测。
局限
机制解释依赖具体模型，跨模型泛化要谨慎。
arXiv 2606.28128具身世界模型

PhysisForcing

研究问题
视频模型常在接触、变形和轨迹连续性上违反物理直觉。
核心方法
用像素轨迹对齐和语义关系对齐，把监督集中在物理一致性区域。
为什么重要
世界模型更可信，机器人就能先在模型里试动作。
局限
从仿真 benchmark 到真实机器人仍有距离。
arXiv 2606.28279代码 agent

HORIZON

研究问题
硬件设计能否像软件仓库一样被 agent 长期演化、评测和回放。
核心方法
用 Markdown harness 生成项目包，包含知识、评测、通过条件和 runtime 策略。
为什么重要
agent 的工程边界在仓库、评测和回放闭环，不在单段代码。
局限
受控代理任务不等于真实芯片设计已被解决。
arXiv 2606.28116大模型训练

Mechanism-Driven Monitors

研究问题
训练故障在 loss 发散前可能已经积累很久。
核心方法
从 flash attention、MoE router 等模块构造早期报警指标。
为什么重要
减少训练浪费，让大模型训练更像工业过程控制。
局限
不同架构的早期故障信号不同，工程部署要结合具体训练栈。

ToolPrivacyBench

研究问题
工具型 agent 可能完成任务，却把不必要的私人信息传给错误工具。
核心方法
构造 2,150 个案例，记录工具参数和后端审计日志。
为什么重要
把 agent 评测从答案是否正确推进到执行轨迹是否合规。
局限
仍是 mock backend 和 benchmark，真实企业系统更复杂。

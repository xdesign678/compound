审计概览

Compound 已是一个设计基础扎实的产品（温暖文学风、书籍品质排版、完整离线架构），但在导航稳定性和 CSS 设计系统落地两个方面存在系统性问题。7 个 Critical 中有 4 个是导航/交互 Bug，3 个是 CSS 架构债；修复后产品体验将从"好用"跃升至"优秀"。

Top 5 最有影响力的改进

1
修复首页自动重定向 + Escape 键跳转 bug — 用户无法稳定停留在首页，加载 2-3 秒后跳转到 /sync 或 /recap，严重破坏基础使用体验。
2
消除 CSS !important 战争 — globals.css 中 111 处 !important 覆盖，是技术债的核心。同一个元素的 border-radius 被 4 处声明覆盖 (18px→8px→12px→16px)。
3
Token 化 CSS 硬编码值 — font-size token 使用率仅 12%（465 处声明中只有 57 处用了变量），spacing 仅 1.2%。设计系统形同虚设。
4
统一认证错误处理 — /sync 和 /review 页面暴露原始英文 "Unauthorized" 文本，无样式、无说明、无操作指引。
5
修复移动端搜索按钮 — 点击搜索图标后跳转到 /offline（离线页面）而非打开搜索，核心功能路径错误。

Critical 问题 7 条

Critical C-01 首页频繁自动重定向到其他页面
H3 用户控制H7 灵活高效Desktop + Mobile
加载首页 2-3 秒后自动跳转到 /sync、/recap 或 /review，行为不确定且不可预测，用户完全无法稳定停留在首页。
去除自动导航逻辑，或限定为仅在明确条件下触发并给用户选择权。文件：app/page.tsx
Critical C-02 Escape 键触发意外跳转到 /sync
H3 用户控制Desktop
在首页按 Escape 键（无论场景），页面会意外导航到 /sync。用户频繁丢失当前工作上下文。
检查 Escape 键事件处理链，确保只在有弹窗/overlay 打开时触发关闭。文件：lib/hooks/useKeyboardShortcuts.ts
Critical C-03 移动端搜索按钮跳转到离线页面
H2 现实匹配Mobile 375
点击 Header 搜索图标后跳转到 /offline（离线模式页面），而非打开搜索面板。核心功能路径完全错误。
修复搜索按钮的 onClick 事件绑定。文件：components/Header.tsx
Critical C-04 /sync 和 /review 暴露原始 "Unauthorized"
H1 状态可见H9 错误恢复Desktop + Mobile
页面顶部显示裸露的英文 "Unauthorized" 文本，无样式、无说明、无操作指引。代码中已有 ERROR_MESSAGES['401'] 映射但未接入。
使用已有的错误映射显示中文消息，添加"前往设置配置 Token"操作按钮。文件：app/sync/page.tsx
Critical C-05 CSS !important 样式战争 (111 处)
H4 一致性globals.css
"CLAUDE DESIGN SYSTEM OVERRIDES" 段使用 111 个 !important 强制覆盖前面的样式。.msg-user 的 border-radius 被 4 处声明覆盖 (18px→8px→12px→16px)。
将 OVERRIDES 段的值合并回组件原始定义，使用 CSS @layer 替代。目标：!important < 15 处。
Critical C-06 font-size Token 使用率仅 12%
H4 一致性globals.css 全文
定义了完整的 8 级 type-scale token，但 465 处 font-size 声明中仅 57 处使用了 token。其余 408 处硬编码像素值，含 42 处不在 scale 上的半像素值。
批量替换 12px→var(--text-xs)、13px→var(--text-sm)、15px→var(--text-base)。
Critical C-07 Cmd+K 命令面板触发 JS 崩溃
H5 错误预防Desktop
从概念详情页返回后按 Meta+k 触发全页面错误 "页面遇到问题"，显示 Sentry 错误。快捷键不应导致页面崩溃。
检查命令面板初始化逻辑的 null safety。文件：components/CommandPalette.tsx

Major 问题 19 条

MajorM-01 "活动" Tab 名称与内容严重不匹配
侧边栏点击"活动"→跳到 /review；tabpanel 显示"健康检查"。"活动"让用户联想到 activity feed，但实际是运维功能。
MajorM-02 Tab 切换状态不一致
首次点击"问答"后页面仍显示 Wiki 内容；selected/active 属性同时出现在两个 Tab 上。
MajorM-03 桌面端导航布局不一致
Wiki Tab 用侧边栏导航，资料/问答 Tab 切换到底部 Tab 栏+顶部 Header，用户需记忆不同页面的导航位置。
MajorM-04 搜索输入触发页面崩溃
某些情况下在搜索框输入文字会触发 JavaScript 错误，显示错误边界。搜索是核心功能路径。
MajorM-05 概念详情页 URL 直链 404
某些交互路径产生 /concept/c-seed-2 URL，但该路由不存在。概念无法通过 URL 分享。
MajorM-06 ~ M-10 CSS Token 使用率极低
Spacing token 使用率 1.2%（859 处中仅 10 处用变量）；114+ 处硬编码 rgba 颜色值；border-radius 11+ 种值碎片化；10+ 种媒体查询断点；79 处硬编码 transition。
MajorM-11 视口切换时 Tab 状态丢失
Desktop 选中 Wiki Tab 后 resize 到 768px，页面变成"问答"Tab。Tab 状态不跟随布局持久化。
MajorM-12 Toast 系统在主页未挂载
DOM 中无 toast 容器元素，操作反馈可能在某些路由上丢失。
MajorSM-01 概念详情无编辑/删除操作
AI 生成的概念用户完全无法修改或删除，缺乏用户控制感。
MajorSM-02 桌面分栏无关闭按钮
概念详情分栏显示没有 X 按钮，新用户不知道如何关闭。
MajorSM-03 ~ SM-07 错误恢复 + 移动端补充
问答失败缺"重试"按钮；无自定义 404 页面；Ask 暴露 JSON 错误；分类横向滚动无视觉暗示；Recap 返回按钮仅 36px。

CSS 设计系统重构路线

globals.css 共 13,124 行、1,656 个选择器。设计 Token 定义完整但执行率极低——这是所有视觉一致性问题的根因。

消除技术债 — 合并 DESIGN SYSTEM OVERRIDES 段（111 个 !important），使用 CSS @layer 替代；清理 17 个未用变量
font-size Token 化 — 408 处硬编码 → var(--text-*)，目标使用率 80%+
spacing Token 化 — 849 处硬编码 → var(--space-*)，目标使用率 60%+
border-radius 统一 — 11 种值 → 4 级 token（4/8/12/999px），7.5px 统一为 8px
rgba 颜色提取 — 114+ 处硬编码 → 语义 CSS 变量，暗色模式自动适配
断点/transition/shadow 标准化 — 统一为 3-4 级标准断点；motion token 替换；shadow scale 扩展
